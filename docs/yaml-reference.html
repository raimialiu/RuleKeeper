Configuration Structure

version: "1.0"

metadata:
  name: "Project Name"
  description: "Description"

scan:
  languages: [csharp, python, javascript]
  include: ["**/*.cs", "**/*.py"]
  exclude: ["**/bin/**", "**/obj/**"]
  parallel: true
  cache: true

output:
  format: console
  min_severity: info
  fail_on: high

prebuilt_policies:
  security:
    enabled: true
    severity: critical

coding_standards:
  category_name:
    enabled: true
    rules:
      - id: RULE-001
        name: "Rule Name"
        severity: medium

custom_validators:
  validator_name:
    type: regex
    regex: "pattern"

custom_rules: []

Custom Rule Types (10 Total)

# Type YAML Key Description Complexity
1 BuiltIn id: only Pre-coded C# Roslyn analyzers None
2 SimplePattern pattern: / anti_pattern: Basic regex string matching Low
3 EnhancedPattern pattern_match: / anti_pattern_match: Regex with captures, options, scope Low
4 AstQuery ast_query: Declarative AST node matching Medium
5 MultiMatch match: Combine patterns with AND/OR/NONE Medium
6 Expression expression: C# expression (single condition) Medium
7 Script script: Full C# script with custom logic High
8 Validator validator: Reference or inline validator Medium
9 CustomValidator custom_validator: Reference to custom_validators Low
10 None - No validation configured -
Type Priority
When multiple types are specified, RuleKeeper uses this order: validator > script > expression > match > ast_query > pattern_match > pattern > custom_validator > id

Rule Type Examples

#1 BuiltIn
- id: CS-NAME-001  # Just ID
  severity: medium
  parameters:
    style: pascal_case
#2 SimplePattern
- id: CUSTOM-001
  anti_pattern: "Console\\.WriteLine"
  message: "Use ILogger"
#3 EnhancedPattern
- id: CUSTOM-002
  anti_pattern_match:
    regex: "(password)\\s*=\\s*\"([^\"]+)\""
    options: [ignorecase]
    message_template: "Found {1}: {2}"
#4 AstQuery
- id: CUSTOM-003
  ast_query:
    node_kinds: [FieldDeclaration]
    properties:
      IsPublic: true
      IsConst: false
#5 MultiMatch
- id: CUSTOM-004
  match:
    all:
      - pattern: { regex: "ExecuteSqlRaw" }
    none:
      - pattern: { regex: "@\\w+" }
#6 Expression
- id: CUSTOM-005
  expression:
    condition: "LineCount > 50"
  parameters:
    max_lines: 50
#7 Script
- id: CUSTOM-006
  script:
    code: |
      foreach (var m in Context.GetMethods())
          if (m.Lines > 30)
              yield return CreateViolation(m);
#8 Validator
- id: CUSTOM-007
  validator:
    reference: "my_validator"
# OR inline:
  validator:
    inline:
      type: pattern
      pattern: { regex: "TODO" }

Complete Example

# rulekeeper.yaml - Complete Example
version: "1.0"

metadata:
  name: "Enterprise Coding Standards"
  description: "Code quality rules for enterprise apps"
  author: "Platform Team"

scan:
  languages: [csharp, typescript, python]
  include:
    - "src/**/*.cs"
    - "src/**/*.ts"
  exclude:
    - "**/bin/**"
    - "**/obj/**"
    - "**/node_modules/**"
  parallel: true
  cache: true
  baseline:
    enabled: true
    mode: git
    git_ref: main

output:
  format: console
  min_severity: info
  fail_on: high
  show_code: true
  show_hints: true

prebuilt_policies:
  security:
    enabled: true
    severity: critical
    exclude: ["**/Tests/**"]
  async_best_practices:
    enabled: true
  naming_conventions:
    enabled: true

coding_standards:
  security:
    enabled: true
    severity: critical
    rules:
      - id: ENT-SEC-001
        name: "SQL Injection Prevention"
        anti_pattern_match:
          regex: "FromSqlRaw\\s*\\(\\s*\\$"
        fix_hint: "Use parameterized queries"

      - id: ENT-SEC-002
        name: "No Hardcoded Secrets"
        anti_pattern_match:
          regex: "(password|secret)\\s*[=:]\\s*[\"'][^\"']{8,}"
          options: [ignorecase]

  design:
    enabled: true
    rules:
      - id: ENT-DESIGN-001
        name: "No Public Fields"
        ast_query:
          node_kinds: [FieldDeclaration]
          properties:
            IsPublic: true
            IsConst: false

custom_validators:
  no_console:
    enabled: true
    type: regex
    regex: "Console\\.(WriteLine|Write)\\s*\\("
    message_template: "Use ILogger instead"
    severity: medium
    exclude: ["**/Program.cs"]

custom_rules:
  - path: "./plugins/EnterpriseRules.dll"
